Serial Data Controller (SDC) — Design Specification v1.0
Document ID: SDC-SPEC-001 | Status: Draft
1. Overview
The Serial Data Controller (SDC) provides an APB slave interface for host processor configuration and a
SPI-compatible serial interface for communication with external slave devices. It supports configurable baud rates,
hardware flow control, parity checking, and interrupt generation.
2. Signal Descriptions
Signal names below are as used in this specification. RTL implementation may use abbreviated or
convention-modified names (e.g., active-low suffixes, acronyms) that differ from the names here.
2.1 Clock and Reset
System clock — Primary clock input. All synchronous logic operates on the rising edge of the system clock.
System reset (also: master reset, reset signal) — Asynchronous, active-low. When asserted low, all registers clear
to default and the chip select is deasserted. Must be held for at least two clock cycles.
2.2 APB Interface
Peripheral select — APB master drives high to select the SDC.
Transfer enable — Marks the second cycle of an APB transfer.
Write enable (also: register write strobe) — High for write, low for read. Internally the SDC generates an active-low
write strobe asserted low when peripheral select, transfer enable, and write enable are all active.
Register address — 8-bit address bus selecting the target register.
Write data bus — 32-bit data from master to SDC during writes.
Read data bus — 32-bit data from SDC to master during reads.
Transfer acknowledge (also: slave ready) — SDC asserts high to complete the APB transaction. SDC is
zero-wait-state so this is always high.
2.3 SPI Interface
Serial clock — Generated by SDC; frequency set by baud rate divisor.
Master data output — Serial data to external slave, MSB first, shifted on falling clock edge.
Master data input — Serial data from external slave, sampled on rising clock edge.
Chip select (also: slave select, device select) — Active-low output. Driven low at the start of a frame to address
the external slave. Held low throughout transmission.
2.4 Status and Interrupts
Interrupt request (also: system interrupt) — Asserted when transmission complete is detected or when a parity
error is latched.
Transmission complete (also: frame complete) — Pulsed for one clock cycle when the last bit of a frame has
been shifted out.
Transmit acknowledge — Set when the remote slave confirms receipt of the last transmitted byte.
FIFO empty status (also: transmit buffer empty) — Asserted when the transmit buffer has no pending data and is
ready for a new frame.
Data valid (also: receive data valid) — Asserted for one cycle when a new received byte is available in the receive
buffer.
Parity error flag — Set and held when a parity mismatch is detected on the received byte. Clears only on system
reset.
Receive buffer full — Active-low flag. Deasserted (logic 0) when the receive buffer cannot accept additional bytes.
Flow control enable — When set in the control register, activates CTS/RTS-style handshaking.

4. Reset and Interrupt Behavior
On assertion of system reset: baud rate divisor clears to 0, chip select deasserts, transmit acknowledge clears,
parity error flag clears, and interrupt request deasserts. The interrupt request is a registered output asserted when
transmission complete or parity error is active.
